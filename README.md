# 🏢 Enterprise Overview

The **Enterprise Multi-Region E-Commerce Platform on AWS** is a production-ready, cloud-native application designed to demonstrate modern DevOps and AWS best practices. It leverages **Amazon EKS**, **Terraform**, **Docker**, **Helm**, **GitHub Actions**, and **ArgoCD** to automate infrastructure provisioning, application deployment, and operations across multiple AWS Regions.

The platform is built with a **microservices architecture**, ensuring high availability, scalability, security, and resilience through Infrastructure as Code (IaC), GitOps, CI/CD automation, and comprehensive monitoring.

---


# 🎯 What This Project Demonstrates

- 🌍 Designing a **Multi-Region AWS** architecture for high availability and disaster recovery.
- ☸️ Deploying and managing **containerized microservices** on Amazon EKS.
- 🏗️ Automating cloud infrastructure using **Terraform (Infrastructure as Code)**.
- 🚀 Building an end-to-end **CI/CD pipeline** with GitHub Actions.
- 🔄 Implementing **GitOps continuous deployment** using ArgoCD.
- 🐳 Packaging applications with **Docker** and managing deployments with **Helm**.
- 🔒 Applying **enterprise security** using IAM, WAF, Secrets Manager, and KMS.
- 📊 Enabling **monitoring and observability** with Prometheus, Grafana, Loki, Tempo, and CloudWatch.
- ⚡ Creating a **scalable, resilient, and production-ready** cloud-native platform following modern DevOps best practices.

# ⚙️ Local Development Setup

Follow the steps below to run the Enterprise Multi-Region E-Commerce Platform locally for development and testing.

---

## 📋 Prerequisites

Ensure the following tools are installed on your system:

| Tool | Version |
|------|----------|
| 🐳 Docker | Latest |
| ☸️ Kubernetes (Kind/Minikube) | Latest |
| 📦 Helm | v3+ |
| 🌍 AWS CLI | v2+ |
| 🏗 Terraform | v1.6+ |
| 🔧 kubectl | Latest |
| 🔀 Git | Latest |
| 🐍 Python | 3.11+ |

---

## 1️⃣ Clone the Repository

```bash
git clone https://github.com/your-username/Enterprise-Multi-Region-E-Commerce-Platform-on-AWS.git

cd Enterprise-Multi-Region-E-Commerce-Platform-on-AWS
```

---

## 2️⃣ Configure AWS Credentials

```bash
aws configure
```

Provide:

```text
AWS Access Key ID
AWS Secret Access Key
Default Region (e.g. ap-south-1)
Output Format (json)
```

Verify

```bash
aws sts get-caller-identity
```

---

## 3️⃣ Create Python Virtual Environment

```bash
python -m venv .venv
```

### Windows

```bash
.venv\Scripts\activate
```

### Linux / macOS

```bash
source .venv/bin/activate
```

Install dependencies

```bash
pip install -r requirements.txt
```

---

## 4️⃣ Start Docker

Verify Docker installation

```bash
docker --version
docker compose version
```

Start Docker Desktop (Windows/macOS) or Docker daemon (Linux).

---

## 5️⃣ Start Local Kubernetes Cluster

### Using Minikube

```bash
minikube start
```

OR

### Using Kind

```bash
kind create cluster --name ecommerce
```

Verify

```bash
kubectl get nodes
```

---

## 6️⃣ Install Helm Dependencies

```bash
helm dependency update ./helm/ecommerce
```

---

## 7️⃣ Deploy Monitoring Stack

```bash
kubectl apply -f kubernetes/monitoring/
```

Verify

```bash
kubectl get pods -n monitoring
```

---

## 8️⃣ Deploy the Application

Using Helm

```bash
helm install ecommerce ./helm/ecommerce
```

OR

Using Kubernetes manifests

```bash
kubectl apply -f kubernetes/
```

---

## 9️⃣ Verify Deployment

```bash
kubectl get all
```

Check Pods

```bash
kubectl get pods
```

Check Services

```bash
kubectl get svc
```

---

## 🔟 Access the Application

Port Forward

```bash
kubectl port-forward svc/frontend 8080:80
```

Open your browser

```
http://localhost:8080
```

---

## 📊 Access Monitoring Dashboards

### Grafana

```
http://localhost:3000
```

### Prometheus

```
http://localhost:9090
```

---

## 🧪 Run Tests

```bash
pytest
```

Run specific tests

```bash
pytest tests/
```

---

## 🛑 Stop the Application

```bash
helm uninstall ecommerce
```

or

```bash
kubectl delete -f kubernetes/
```

---

## 🧹 Clean Up

Delete local cluster

### Minikube

```bash
minikube delete
```

### Kind

```bash
kind delete cluster --name ecommerce
```
